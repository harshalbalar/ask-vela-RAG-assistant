# Postmortem INC-2025-048: auth-service — login failures spiked to 30%

**Date:** 2025-04-15 · **Service:** `auth-service` · **Severity:** SEV2 · **Doc ID:** INC-2025-048

## Summary
On 2025-04-15, `auth-service` experienced login failures spiked to 30%.

## Root cause
The root cause was an expired Okta signing certificate.

## Resolution
The team rotated the certificate and added an expiry alert. Mitigation followed the incident runbook (ENG-INC-009).

## Action items
- Add or improve monitoring to catch this earlier.
- Document the failure mode in the `auth-service` runbook.
- Review related services for the same risk.

This was a blameless postmortem reviewed with the owning team.
