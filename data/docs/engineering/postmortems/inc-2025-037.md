# Postmortem INC-2025-037: ingest-api — 5% of events shed for 22 minutes

**Date:** 2025-03-19 · **Service:** `ingest-api` · **Severity:** SEV2 · **Doc ID:** INC-2025-037

## Summary
On 2025-03-19, `ingest-api` experienced 5% of events shed for 22 minutes.

## Root cause
The root cause was an undersized ingest buffer during a traffic spike.

## Resolution
The team increased buffer size and added autoscaling. Mitigation followed the incident runbook (ENG-INC-009).

## Action items
- Add or improve monitoring to catch this earlier.
- Document the failure mode in the `ingest-api` runbook.
- Review related services for the same risk.

This was a blameless postmortem reviewed with the owning team.
