# Postmortem INC-2025-051: stream-processor — Kafka consumer lag grew to 9 minutes

**Date:** 2025-04-28 · **Service:** `stream-processor` · **Severity:** SEV2 · **Doc ID:** INC-2025-051

## Summary
On 2025-04-28, `stream-processor` experienced Kafka consumer lag grew to 9 minutes.

## Root cause
The root cause was a poison-pill event blocking a partition.

## Resolution
The team added a dead-letter queue for malformed events. Mitigation followed the incident runbook (ENG-INC-009).

## Action items
- Add or improve monitoring to catch this earlier.
- Document the failure mode in the `stream-processor` runbook.
- Review related services for the same risk.

This was a blameless postmortem reviewed with the owning team.
