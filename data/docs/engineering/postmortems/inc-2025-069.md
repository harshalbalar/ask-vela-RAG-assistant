# Postmortem INC-2025-069: notification-service — alert emails delayed up to 18 minutes

**Date:** 2025-06-09 · **Service:** `notification-service` · **Severity:** SEV2 · **Doc ID:** INC-2025-069

## Summary
On 2025-06-09, `notification-service` experienced alert emails delayed up to 18 minutes.

## Root cause
The root cause was a backed-up notification queue after a provider outage.

## Resolution
The team added a fallback email provider. Mitigation followed the incident runbook (ENG-INC-009).

## Action items
- Add or improve monitoring to catch this earlier.
- Document the failure mode in the `notification-service` runbook.
- Review related services for the same risk.

This was a blameless postmortem reviewed with the owning team.
